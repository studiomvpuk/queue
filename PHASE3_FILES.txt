KEY FEATURES BY MODULE
======================

PAYMENTS:
- Paystack initialization returns authorizationUrl + reference (valid ~30 min)
- Webhook verification: HMAC-SHA512 on raw body
- Idempotent processing by payment reference
- Atomic booking creation on charge.success event
- WS event: payment:success emitted to user
- Never logs card data; never stores authorization URL post-response
- Amount calculation: getPrioritySlotPrice(LocationCategory) → kobo value
- Free for users with isAccessibility=true

PRIORITY SLOTS:
- 20% capacity cap per location per day
- Priority bookings skip non-priority queue members same slot
- Accessibility self-declaration with optional proof document URL
- Audit trail for all declarations and priority booking state changes
- canBookPriority() returns { allowed, remainingSlots }

VIRTUAL CALLS:
- Daily.co integration via API key
- STAFF-only room creation
- Room name: queuease-{booking.code}
- 1-hour room expiry (configurable at Daily.co)
- WS event: virtual:room-ready sent to user with URL + expiry
- Extensible: booking metadata can store roomUrl

API CLIENTS:
- OAuth 2.0 client credentials flow
- Admin creates client → returns clientId + clientSecret (NEVER retrievable again)
- clientSecret hashed with argon2
- Token endpoint: POST /oauth/token → 30-min JWT (type: 'client')
- Scope-based access via @RequireScopes('scope1', 'scope2') decorator
- ApiClientAuthGuard: allows user JWT or client JWT
- Rate limit: 10 requests/min on /oauth/token

WEBHOOKS:
- Clients create subscriptions for events (e.g., 'booking.created', 'booking.served')
- Webhook secret returned once (used for HMAC-SHA256 verification)
- Signature header: X-QueueEase-Signature
- BullMQ-backed dispatch with 3 retries:
  * Exponential backoff: 2s → 4s → 8s
  * 10-second timeout per request
  * Removed on completion (no persistence post-success)
- Idempotent: keyed by (event, bookingId, timestamp)

USSD:
- Feature-gated by USSD_ENABLED env var
- Termii webhook signature verification
- 4-step state machine: category → location → slot → confirm
- State stored in Redis with 5-minute TTL per phone
- Auto-cleanup via Redis TTL
- Returns USSD menu text (CON/END prefix)
